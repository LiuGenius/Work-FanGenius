> #### result tips:


##### 0-26:你在生活中所经历的压力经验不够，或是你没有正确地分析你自己。你最好更主动些，在工作、社交、娱乐等活动上多增加些刺激。做松弛训练对你没有什么用，但找一些辅导也许会有帮助。
##### 27–37:你的生活可能是相当沉闷的，即使刺激或有趣的事情发生了，你也很少作反应。可能你必须参加更多的社会活动或娱乐活动，以增加你的压力激活反应。
##### 38–48:你对所遭遇的压力器很不易为所动，甚至是不当一回事，好像并没有发生过一样。这对你的健康不会有什么负面影响，但你的生活缺乏适度的兴奋，因此趣味也就有限。
##### 49–59:你能够控制你自己的压力反应，你是一个相当放松的人。也许你对于所遇到的各种压力器，并没有将他们解释为威胁，所以你很容易与人相处，可以毫无惧怕地担任工作，也没有失去信心。
##### 60–70:你生活中的兴奋与压力也许是相当适中的。偶尔会有一段时间压力太大，但你也许有能力去享受压力，并且很快回到平衡状态，因此对你的健康不会造成威胁。做一些松弛训练仍是有益的。
##### 71–81:你的压力程度中等，可能正开始对健康不利。你可以仔细反省自己对压力器如何作出反应，并学习在压力器出现时，控制自己的肌肉紧张，以消除生理激活反应。好老师对你有帮助，要不然就选择适合你的肌肉松弛录音带。
##### 82–92:你正经历太多的压力，这正在损害你的健康，并令你的人际关系发生问题。你的行为会伤害自己，也会影响其他人。因此，对你来说，学会如何减除自己的压力反应是非常必要的。你可能必须花时间做练习，学习控制压力，也可以寻求专业的帮助。
##### >93:你确实正以极度的压力反应伤害你自己的健康，你需要专业心理治疗师给与一些忠告，它可以帮助你削减你对压力器的知觉，并帮助你改良生活的品质。

## 接口需求

- #### 1.获取所有设备信息
	- ##### 字段
		- ##### a.所属机构ID
		
- #### 2.设备下发指令
	- ##### 字段
		- ##### a.title  (标题)
		- ##### b.SerialNumberIdList (设备列表)
		- ##### c.content (内容)
		- ##### d.msg (信息)

- #### 3.用户登录
	- ##### 字段
		- ##### a.serialNumber  (设备ID)
		- ##### b.account  (账号)
		- ##### c.passWord  (密码)
		
- #### 4.用户注册
	- ##### 字段
		- ##### a.birthday  (出生年月日)
		- ##### b.account  (账号)
		- ##### c.passWord  (密码)
		- ##### c.role  (角色)
		- ##### c.sex  (性别)
		- ##### c.userName  (姓名)
		- ##### c.serialNumber  (设备ID)

- #### 5.更新用户身体信息
	- ##### 字段
		- ##### a.UserBodyInfo
		
- #### 6.添加压力评测报告
	- ##### 字段
		- ##### a.压力评测报告对象
		
- #### 7.创建游戏房间
创建房间后，生成一个websocket连接，加入这个websocket连接视为进入房间。断开连接视为退出房间
	- ##### 字段
		- ##### a.gameName (游戏名称)
		- ##### b.roomName (房间名称)
		
		


```

=======================================websock通信协议======================================


======================================游戏流程协议开始======================================

//流程1：客户端连接websock协议后向服务端发送以下数据
//用户端 TO 服务器
{
	"function":"userJoin"
	"userId":"XXX",//用户ID
	"serialNumber":"123456789sad",//设备序列号
	"roomId":xxxxxx//房间号信息，区分加入的是哪个房间
}

//流程2：每当有新用户加入或者有用户退出，服务端向所有用户端发送以下数据
//服务器 TO 用户端
{
	"function":"userJoin/userOut",
	"userList":[user,user],//用户对象集合，包括当前用户本身
	"roomId":xxxxxx//房间号信息，区分加入的是哪个房间
}

//流程3：游戏开始时，服务端向用户端发送以下数据
//服务器 TO 用户端
{
	"function":"gameStart",
	"roomId":xxxxxx//房间号信息，区分加入的是哪个房间
}

//流程4：房间创建人才可以开始游戏，房间创建人退出房间后房间解散。游戏开始后不需要解散房间。
//用户端 TO 服务器
{
	"function":"gameStart",
	"roomId":xxxxxx//房间号信息，区分加入的是哪个房间
}

//流程5：用户端游戏时，每当进度变更向服务端发送以下数据，收到进度值为100时，代表该用户游戏结束，记录游戏时间和分数
//用户端 TO 服务器
{
	"function":"gameProgress",
	"roomId":xxxxxx,//房间号信息，区分加入的是哪个房间
	"progress":1-100//进度
}

//流程6：当服务端收到客户端游戏进度变更的信息时，向用户端发送以下协议
//服务器 TO 用户端
{
	"function":"gameProgress",
	"data":["user1":50,user2:60,...],
	"roomId":xxxxxx,//房间号信息，区分加入的是哪个房间
}

//流程7：当服务端收到所有用户端游戏进度为完成时，向所有客户端发送以下协议
//ps:当游戏开始后，三分钟时间内还有用户未完成游戏，这个用户数据为 “未完成”，时间为 “--”
//服务器 TO 用户端
{
	"function":"gameOver",
	"roomId":xxxxxx,//房间号信息，区分加入的是哪个房间
	"data":[	
		{
			user:{USER}
			useResult:"完成",
			useTime:"50",//时间(秒)
			userName:"发顺丰",//用户名
			userAvatar:"xxx"//用户头像
		},
		{
			...
		},...
	]
}

======================================游戏流程协议结束======================================


======================================邀请流程协议开始======================================

//流程1：用户邀请其他用户加入游戏时，使用极光推送以下推送自定义消息
{
	"function":"invite",
	"inviteUserName":"范冰冰",//邀请人姓名
	"inviteUserId":xxxxxx//邀请人Id
}

```
